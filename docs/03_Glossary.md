# Glossary

## Purpose

This glossary defines the technical terms and abbreviations used throughout the LegacyLens AI project. It provides a common understanding for developers, project managers, and stakeholders.

---

| Term | Definition |
|------|------------|
| LegacyLens AI | An AI-powered platform designed to analyze legacy software systems and provide modernization insights, recommendations, and documentation. |
| Legacy System | A software system that is still in use but relies on outdated technologies, architectures, or development practices. |
| Software Modernization | The process of improving or updating a legacy system while preserving its business functionality. |
| Project Analysis | The automated examination of a software project to understand its technologies, architecture, dependencies, and overall quality. |
| Technology Stack | The complete set of technologies used in a project, including programming languages, frameworks, databases, and development tools. |
| Dependency | An external library, framework, or package that a software project depends on to function correctly. |
| Architecture | The overall structure of a software system, including its components and how they interact with each other. |
| Static Analysis | The process of analyzing source code and project files without executing the application. |
| Findings | Issues, observations, or potential risks detected automatically during the analysis process. |
| Recommendation | An AI-generated suggestion intended to improve the quality, maintainability, or modernization of the software project. |
| Technical Debt | The future maintenance cost caused by outdated technologies, poor design decisions, or shortcuts taken during development. |
| Documentation | Technical information generated or summarized by the platform to help developers understand the project. |
| Executive Summary | A concise AI-generated overview describing the project's current state, strengths, weaknesses, and modernization opportunities. |
| Roadmap | A prioritized implementation plan that guides the modernization of the analyzed project. |
| AI Assistant | A context-aware conversational assistant that answers questions about the currently analyzed project using the generated analysis. |
| Health Score | An overall indicator representing the maintainability and quality of the analyzed software project. |
| Maintainability | The ease with which a software system can be modified, updated, tested, or extended. |
| Repository | A collection of source code, configuration files, documentation, and project resources managed together. |
| Source Code | The human-readable code written by developers to build the software application. |
| Framework | A software platform that provides predefined components and structure for building applications. |
| Programming Language | A formal language used to write software, such as Python, Java, PHP, or JavaScript. |
| Module | A logical part of a software application responsible for a specific functionality. |
| Project Context | The collection of information generated during analysis (architecture, technologies, findings, documentation, recommendations, etc.) that is provided to the AI Assistant. |
| ZIP Upload | The process of submitting a software project by uploading a compressed ZIP archive. |
| Report | A downloadable document summarizing the analysis results, findings, and recommendations. |
| Dashboard | The main interface displaying an overview of all analyzed projects and their status. |
| Analysis | The workspace where users explore all results related to a selected project. |
| Overview | A summary page presenting the most important information and metrics about the analyzed project. |
| Dependencies | A section listing all detected libraries, frameworks, packages, and their versions. |
| Modernization Readiness | An indicator showing how prepared the project is for modernization based on analysis results. |
| Risk Level | An estimation of the technical risks associated with maintaining or modernizing the project. |
| PDF Report | A downloadable report containing the project's analysis and recommendations in PDF format. |
| MVP (Minimum Viable Product) | The first functional version of LegacyLens AI that includes the essential features needed to validate the project concept. |
